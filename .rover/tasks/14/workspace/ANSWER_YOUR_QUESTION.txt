OPENHANDS:
  What: Framework for autonomously running tasks
  Uses: Local code execution via PowerShell
  Needs .NET SDK: YES (on Windows) ⚠️
  Works now: NO (needs .NET SDK)
  What happens: Claude executes code on your machine
  Example: "Create file" → Creates actual file on your computer

================================================================================
                              WHY .NET SDK?
================================================================================

Flow of OpenHands:

1. User: "Create a Python file and run it"
                    ↓
2. OpenHands → Claude Agent SDK: "What commands should I run?"
                    ↓
3. Claude: "Run these commands: python create_script.py"
                    ↓
4. OpenHands tries to execute commands
                    ↓
5. OpenHands uses Windows PowerShell to execute
                    ↓
6. PowerShell requires .NET SDK to work properly
                    ↓
7. If .NET SDK installed: Commands execute ✓ File created ✓
   If .NET SDK NOT installed: ERROR ✗

BOTTOM LINE:
.NET SDK is needed because OpenHands uses PowerShell,
and PowerShell needs .NET SDK on Windows.

But YOUR Claude Agent SDK doesn't do any of this!
It just talks to Claude API over the internet.
