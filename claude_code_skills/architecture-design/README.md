# Architecture & Design Skills

This category contains skills for system architecture patterns, microservices, API design, and cloud architecture.

## Skills in This Category

### `api-design-principles`
Master REST and GraphQL API design principles to build intuitive, scalable, and maintainable APIs that delight developers.

**Use when:**
- Designing new APIs
- Reviewing API specifications
- Establishing API design standards

**Key topics:**
- REST API design
- GraphQL API design
- API best practices
- Scalability
- Maintainability

---

### `architecture-patterns`
Implement proven backend architecture patterns including Clean Architecture, Hexagonal Architecture, and Domain-Driven Design.

**Use when:**
- Architecting complex backend systems
- Refactoring existing applications
- Improving code maintainability

**Key topics:**
- Clean Architecture
- Hexagonal Architecture
- Domain-Driven Design (DDD)
- Separation of concerns
- Dependency management

---

### `microservices-patterns`
Design microservices architectures with service boundaries, event-driven communication, and resilience patterns.

**Use when:**
- Building distributed systems
- Decomposing monoliths
- Implementing microservices

**Key topics:**
- Service boundaries
- Event-driven architecture
- Resilience patterns
- Service communication
- Distributed systems

---

### `multi-cloud-architecture`
Design multi-cloud architectures using a decision framework to select and integrate services across AWS, Azure, and GCP.

**Use when:**
- Building multi-cloud systems
- Avoiding vendor lock-in
- Leveraging best-of-breed services from multiple providers

**Key topics:**
- Multi-cloud strategy
- Service selection
- Cloud integration
- Vendor lock-in avoidance
- AWS, Azure, GCP

---

## Common Use Cases

### Designing a New System
Combine these skills:
1. `architecture-patterns` - Overall architecture
2. `api-design-principles` - API design
3. `microservices-patterns` (if distributed) - Service design

### Building a Microservices Architecture
Use:
1. `microservices-patterns` - Service design
2. `api-design-principles` - API design
3. `architecture-patterns` - Internal service architecture
4. Consider also: `deployment-pipeline-design` from DevOps & Infrastructure

### API-First Development
Use:
1. `api-design-principles` - API design
2. `architecture-patterns` - Backend architecture
3. Consider also: Backend or Frontend skills based on implementation

### Multi-Cloud Strategy
Use:
1. `multi-cloud-architecture` - Cloud strategy
2. `architecture-patterns` - System architecture
3. Consider also: `hybrid-cloud-networking` from DevOps & Infrastructure
