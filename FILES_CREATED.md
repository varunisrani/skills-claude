## File Organization Principles

This project follows these organization principles:

1. **Layered Architecture**: Clear separation between API, Service, Repository, and Model layers
2. **Single Responsibility**: Each file has one clear purpose
3. **Dependency Direction**: Dependencies flow inward (API → Service → Repository → Model)
4. **Type Safety**: Type hints in every file
5. **Testability**: Each layer can be tested independently
6. **Documentation**: Each file has clear docstrings
7. **Conventions**: Consistent naming and structure throughout
